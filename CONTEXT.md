# Domain Glossary — Reverse HH

## Core concepts

| Term | Definition |
|------|------------|
| **User** | Единый аккаунт. Может иметь профиль соискателя и/или участвовать в компаниях. |
| **Candidate profile** | Профиль соискателя, привязанный к User. |
| **Resume** | Основная сущность каталога. У одного соискателя может быть несколько резюме. Имеет опубликованную версию и черновик. |
| **Resume snapshot** | Зафиксированная копия резюме на момент отправки отклика. |
| **Test** | Набор вопросов, привязанный к конкретному резюме. Если есть — обязателен для отклика. |
| **Company** | Работодатель. Проходит верификацию. Может быть архивирована. |
| **Vacancy** | Непубличная вакансия компании. Статусы: `draft`, `active`, `archived`. |
| **Application** | Отклик работодателя на резюме конкретной вакансией. Центральная сущность домена. |
| **Chat** | Открывается после `accepted`. Только текст и ссылки. |
| **Complaint** | Жалоба на компанию, сообщение, отклик или резюме. |
| **Audit log** | Внутренний журнал значимых событий для поддержки и безопасности. |

## Application statuses

`sent` → `viewed` → `accepted` | `rejected` | `auto_rejected` | `expired` → `reactivation_requested` → `reactivated` | `closed_after_acceptance`

## Resume visibility

- `public` — всем верифицированным работодателям в каталоге
- `link_only` — только по прямой ссылке
- `hidden` — никому

Дополнительно: скрытие от конкретных компаний/доменов/ИНН, блокировка компании.

## Company roles

- `owner` — управление компанией, верификацией, участниками, вакансиями
- `recruiter` — вакансии, отклики, чаты (свои + по назначенным вакансиям)

## Work formats

`remote`, `office`, `hybrid`, `field`, `relocation`, `shift`

## Auto-reject conditions (MVP)

- зарплата ниже ожиданий
- неподходящий формат работы
- неподходящий город/релокация

Режимы: показывать с пометкой или автоматически отклонять (лимит списывается).

## Key business rules

1. Одна пара `resume_id + vacancy_id` — один отклик навсегда.
2. Разные вакансии одной компании на одно резюме — без ограничений.
3. Лимит откликов списывается при отправке, не при принятии.
4. Контакты кандидата не раскрываются автоматически после принятия.
5. При блокировке компании чаты становятся read-only.
6. Работодатель не может отозвать отправленный отклик.
7. При редактировании теста резюме скрывается до публикации.

## Avoid

- Не называй Application «откликом кандидата» — откликается работодатель.
- Не публикуй вакансии в каталог — вакансии непубличные.
- Не используй «уровень опыта» (junior/middle/senior) — вместо этого блок опыта работы.
